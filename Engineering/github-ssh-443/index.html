SSH通过443端口连接GitHub

GitHub提供了两种协议供用户使用Git连接——SSH和HTTPS。理论上我可以随意选择两者之一连接到我在GitHub上的代码仓库，无论是将云端的仓库clone到本地，还是将本地的修改push到云端。然而，出于一些奇奇怪怪的原因，我所在的办公网络环境禁止了22端口，而22端口正是GitHub提供SSH访问的端口号。尽管可以换用HTTPS协议，但无论如何将我电脑上的所有代码仓库的上游都从git@github.com:...修改称https://github.com/...仍然是一个繁重的体力活。

为了一劳永逸地解决这个问题，最理想的解决方式是让Git的SSH协议改用22以外的其他端口连接GitHub。

如此一来，ssh.github.com就成为了github.com的“冒名顶替”者。当Git通过SSH协议试图访问github.com的时候，SSH会发现它是ssh.github.com的别名，因此会用443端口实际连接到后者。这样，就绕开了本地网路环境对22端口的限制。

（我之所以使用上述方法，是因为我的网络情况比较复杂。建议公司的朋友在使用暗渡陈仓之策前，先和IT部门确认下是否合规。）
